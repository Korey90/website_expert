# 💼 Software House – Full Business Playbook

---

## 1️⃣ Negotiation-Ready Clauses

**Liability:**

```text
Total liability of the Contractor shall not exceed:
(a) 100% of total fees paid; or
(b) £50,000
whichever is lower.
```

**IP Protection:**

```text
Nothing prevents the Contractor from using know-how, techniques, or reusable components developed during the Project in other projects.
```

**Payment Protection:**

```text
Contractor reserves the right to suspend work immediately in case of overdue payments without liability for delays.
```

**Scope Protection:**

```text
Any work not explicitly included in the Statement of Work shall be treated as a Change Request.
```

**Acceptance Fallback:**

```text
If the Client does not respond within 5 working days, the deliverable shall be deemed accepted.
```

**SLA Limitation:**

```text
SLA applies only if explicitly agreed in writing and paid as a separate service.
```

**Indemnity:**

```text
The Client agrees to indemnify the Contractor against any claims arising from materials provided by the Client.
```

---

## 2️⃣ Checklist Negocjacyjna

* **Finanse:** zaliczka, milestone payments, prawo do zatrzymania pracy
* **Liability:** limit, brak unlimited liability
* **Scope:** zamknięty, Change Requests płatne
* **IP:** zachowujesz reusable code
* **Security / GDPR:** tylko reasonable efforts
* **SLA:** płatne, nie 24/7 free
* **Exit:** możliwość zakończenia współpracy + zapłata za wykonaną pracę

---

## 3️⃣ Red Flags & How to Handle

| Red Flag                      | Jak odpowiadać                                                           |
| ----------------------------- | ------------------------------------------------------------------------ |
| Unlimited liability           | “Liability aligned with project value, unlimited exposure is not viable” |
| Full IP transfer              | “We transfer final product rights, retain reusable components”           |
| Undefined scope + fixed price | “Fixed price for defined scope, extras via Change Requests”              |
| Free support forever          | “30-day warranty included, ongoing support via maintenance plan”         |
| Client controls timeline      | “Timelines depend on timely feedback, delays extend dates”               |
| Penalties tylko dla Ciebie    | “Penalties should apply fairly to both parties”                          |
| Ownership before payment      | “Ownership transfers upon full payment, standard practice”               |

## 4️⃣ Pricing Model – Laravel + React

### Modele

1. Fixed Price – małe / średnie projekty
2. Time & Materials – SaaS, custom systems
3. Value-Based – projekty generujące revenue

### Wycenianie krok po kroku

1. Rozbij projekt na moduły (frontend, backend, auth, API)
2. Estymacja godzin + buffer 20–40%
3. Stawka godzinowa: £60–100/h dla agency
4. Final Price = (estymacja × stawka) × value multiplier
5. Pakiety: Starter, Growth, Pro
6. Discovery Phase: płatne £500–£2k, odliczane od projektu

---

### Przykład:

| Moduł            | Godziny   |
| ---------------- | --------- |
| Frontend         | 40        |
| Backend          | 50        |
| Admin panel      | 25        |
| Integracje       | 20        |
| QA / fixes       | 15        |
| **Total**        | 150       |
| Buffer           | +50 → 200 |
| Stawka           | £80/h     |
| **Cena bazowa**  | £16,000   |
| Value Multiplier | ×1.3      |
| **Final Price**  | £20,800   |

## 5️⃣ Skrypt Sprzedażowy

### 5.1 Otwarcie

> “Dzięki za kontakt. Zanim przejdziemy do wyceny, chcę dobrze zrozumieć Twój biznes i cel projektu.”

---

### 5.2 Discovery

* Co dokładnie system ma robić?
* Jaki jest główny cel projektu?
* Jak projekt generuje przychody lub oszczędności?
* Priorytet: czas / budżet / jakość?

---

### 5.3 Reframe

> “Rozumiem, że główny problem to X, a celem Y. Proponuję system, który nie tylko spełnia wymagania, ale automatyzuje i zwiększa revenue.”

---

### 5.4 Propozycja Rozwiązania

**Opcja A:** podstawowe, szybkie wdrożenie
**Opcja B:** skalowalne, Laravel + React + dashboard + API

> Sprzedajesz opcję B jako rozsądną inwestycję.

---

### 5.5 Cena

* Technika “range + confidence”

> £12k–£20k, bliżej £18k z powodu custom dashboard i integracji

---

### 5.6 Zamknięcie

> “Jeżeli ma to sens, kolejnym krokiem jest przygotowanie dokładnej specyfikacji i start projektu.”

---

### 5.7 Objection Handling

* Za drogo → “Czy problemem jest budżet czy zakres?”
* Muszę się zastanowić → “Co dokładnie potrzebujesz przemyśleć?”
* Mam tańszą ofertę → “Czy porównujemy to samo rozwiązanie, czy tylko cenę?”

---

### 5.8 Discovery Phase jako produkt

> “Możemy zacząć od discovery: przygotuję architekturę i plan projektu, koszt £X, odliczany od projektu.”
