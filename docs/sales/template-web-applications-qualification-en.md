## Qualification Criteria

### Must-Have
- [ ] Budget ≥ £5,999 acknowledged (or £599 for paid discovery phase)
- [ ] Problem is specific: can describe the workflow to be replaced or built
- [ ] Decision-maker identified and engaged
- [ ] Minimum 8–12 weeks timeline accepted
- [ ] No expectation of guaranteed outcomes or revenue guarantees

### Nice-to-Have
- [ ] Prior failed attempt with off-the-shelf software (creates urgency)
- [ ] Internal technical resource available for handover
- [ ] Clear monetisation model (especially for SaaS)
- [ ] Existing wireframes, user stories, or requirements document
- [ ] Budget for ongoing maintenance retainer post-launch

### Red Flags
- 🚨 "I want a LinkedIn clone / Airbnb clone for £10k"
- 🚨 "It's simple — shouldn't take more than a week"
- 🚨 No defined users, no defined outcomes
- 🚨 "I'll figure out the features after it's built"
- 🚨 Client wants to own all IP but also expects ongoing support for free
- 🚨 Expects a fixed price before any discovery phase

---

## Scope and Boundaries
**Entry point:** Paid Discovery Phase from £599 — non-negotiable for any bespoke application.

**Discovery deliverables:** technical specification, user flow diagrams, architecture overview, and a fixed-price quote for the build.

**Scope triggers requiring specialist assessment:**
- Native mobile app requirements (iOS/Android) → outside current service
- AI/ML features → requires specialist sub-contractor
- Real-time video / audio (WebRTC) → separate scoping required
- Data residency or healthcare data compliance → Legal Compliance Agent trigger

## Risks and Dependencies
- Scope not defined → fixed-price quote is impossible without paid discovery
- Multiple decision-makers with conflicting visions → discovery phase must include all key stakeholders
- Third-party API integrations → must be verified as technically feasible before quoting
- Existing codebase → may require full rewrite depending on quality; code audit (£299) strongly recommended

---

## Assumptions
- Paid discovery phase (£599) is sold as the first step for any complex or unclear scope
- Client is aware that the build price is separate from and follows the discovery phase
- Post-launch maintenance is scoped separately
